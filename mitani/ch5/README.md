# アドレスとポインタ
## アドレスとポインタ
### コンピュータのメモリー
* メモリーには、プログラムを実行している途中に扱う、さまざまな情報（データ）が格納されている。
* プログラムを実行すると、メモリーの一部がそのプログラムのために割り当てられ、プログラムで使用される変数の値や関数などの情報が格納される。
* メモリー上に割り当てられる位置を表す値を **アドレス** という。

### アドレス
* これまで、変数の初期化（宣言と代入）は次のように行なっていた。
    ```c
    int i = 5;
    ```
* この命令が実行されると、`int` 型の入れ物が準備され、その中に `5` という値が格納される。
* メモリーに対して、どのような操作が行われるか。
  * `int` 型は4バイトの大きさがある。そのため、`int` 型の変数が宣言されると、メモリーのどこかに4バイトの領域が確保され、その領域が変数 `i` のために割り当てられる。そして、そこに `5` という値を表すデータが格納される。
* 変数 `i` のための領域がメモリーのどこに割り当てられたかは、次のようにして確認できる。
    ```c
    int i = 5;
    printf("変数iのアドレスは %p", &i);
    ```
    * 変数の前に `&` を付けると、その変数のアドレスを知ることができる。
    * アドレスを出力するためには `%p` を使い、メモリー上の位置を16進数表現したものが出力される。
    * 4バイト分の領域を占めているが、**その領域の先頭の位置** を表示する。

### ポインタ
* 変数名に `&` を付けることで、その変数のアドレスを知ることができた。
* アドレスは、次のようにして **ポインタ変数** と呼ばれる変数に代入できる。
    ```c
    int i = 5;
    int *p = &i;
    ```
* ポインタ変数は、アドレスを格納するための変数
* ポインタ変数を宣言するときは、変数名の前に `*` を付ける。`p` という名前の箱が用意されて、そこに `int` 型の変数のアドレスを代入できるようになる。
* 変数 `p` に入るものは、数値ではなくアドレス。アドレスは特定の場所を指し示すものなので、ポインタ変数という。

### ポインタを使った値の参照
* **アドレスがわかることが、何の役に立つか**
  * 変数名を使わずに、ポインタから値にアクセスすることができる。
  * アドレスがわかれば、名前は必要ない。
* 先ほどのコードを例にする。
    ```c
    int i = 5;
    int *p = &i;
    ```
  * ポインタ変数の前に `*` を付けることで、変数 `p` に代入されているアドレスに存在する値（この例では `5`）を参照できるようになる。変数 `i` の名前を使わなくても、ポインタによって変数 `i` の値を参照できるようになる。
  * プログラムの中では、変数名 `i` を使って表記した場合と `*p` と表記した場合の両方とも、メモリー上の同じ値を参照することになる。
* 別の例で確認する。
    ```c
    int i = 5;
    int *p = &i;

    printf("iのアドレスは %p\n", &i);
    printf("pの値は %p\n", &i);
    printf("iの値は %d\n", i);
    printf("*pの値は %d\n", *p);
    ```
  * 実行結果
    ```
    iのアドレスは 0x16b56758c
    pの値は 0x16b56758c
    iの値は 5
    *pの値は 5
    ```
* これまでの、通常の変数とポインタ変数の違いを表にまとめる。
| 宣言 | int型の値を参照するときの表記 | アドレスを参照するときの表記 |
| ---- | ---- | ---- |
| int i; | i | &i |
| int *p; | *p | p |
